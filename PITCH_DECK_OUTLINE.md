## Slide 4: How It Works

### Multi-model AI orchestration in practice.

```
User intent → CoreeyAI router → Optimal model selected → Task executed → Risk-checked → Action taken
```

| Step | Engine | Function |
|------|--------|----------|
| 1. Analysis | Claude (Anthropic) | Deep reasoning, long-context market analysis |
| 2. Real-time data | Grok (xAI) | Live trend analysis, current market signals |
| 3. Verification | Perplexity | Research, cross-reference, knowledge retrieval |
| 4. Execution | gTrade engine | Risk-managed position entry/exit |
| 5. Safety | Hard-coded risk rails | Max 5x leverage, 1% per-trade risk, 0.8% daily loss cap |

**Visual:** Animated flow diagram (or static with numbered steps). Dark background, Electric Cyan (#00D4FF) for AI flows, Signal Gold (#C9A84C) for safety/risk elements.

**Speaker notes:**
"Here's how it actually works. When the system needs to make a decision, CoreeyAI routes to the right model. Claude handles deep reasoning and long-context analysis. Grok processes real-time market data. Perplexity cross-references and verifies. Then the gTrade engine executes — but only after passing through hard-coded risk management. Max five-x leverage. One percent risk per trade. Zero-point-eight percent daily loss cap. Three hundred fifty dollar minimum cash reserve. The safety rails aren't suggestions — they're architecture."
